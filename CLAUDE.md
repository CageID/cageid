## What This Is

CAGE (Confirmed Age, Granted Entry) is an age verification identity provider. The monorepo is in early development — `apps/server` is the core backend; `apps/web` and `apps/docs` are Next.js stubs from the Turborepo scaffold.

## Package Manager & Task Runner

This repo uses **pnpm** (v9) and **Turborepo**. All commands should be run from the repo root unless working in a specific app.

```bash
pnpm install               # install all workspace dependencies
pnpm dev                   # start all apps in parallel
pnpm dev --filter=server   # start only the server (port 3001)
pnpm build                 # build all apps
pnpm lint                  # lint all apps (ESLint, zero warnings allowed)
pnpm check-types           # type-check all apps
```

### Server-specific

```bash
cd apps/server
pnpm dev          # tsx watch src/index.ts — hot reload, no DB required at startup
pnpm build        # tsc → dist/
pnpm start        # node dist/index.js (production)
pnpm db:generate  # drizzle-kit generate (needs DATABASE_URL)
pnpm db:migrate   # drizzle-kit migrate (needs DATABASE_URL)
```

## Environment Variables

Copy `apps/server/.env.example` to `apps/server/.env`:

```
DATABASE_URL=postgres://postgres:postgres@localhost:5432/cageid
PORT=3001          # defaults to 3001 if unset
```

The server starts without `DATABASE_URL` — the DB connection is only established when `src/db/index.ts` is imported by a route, not at startup.

## Architecture

### Monorepo Layout

- **`apps/server`** — Hono API server (the CAGE identity provider backend)
- **`apps/web`** — Next.js frontend (stub, not yet developed)
- **`apps/docs`** — Next.js docs site (stub)
- **`packages/eslint-config`** — shared ESLint configs (`base`, `next-js`, `react-internal`)
- **`packages/typescript-config`** — shared tsconfigs (`base`, `node`, `nextjs`, `react-library`)
- **`packages/ui`** — shared React component library (`@repo/ui`)

### `apps/server` internals

Entry point: `src/index.ts` — creates the Hono app, registers routes, calls `serve()`.

```
src/
  db/
    schema.ts     # Drizzle table definitions (PostgreSQL)
    index.ts      # postgres.js client + drizzle() instance, exported as `db`
  routes/         # Hono route handlers
  services/       # Business logic (called by routes)
  middleware/     # Hono middleware (auth, logging, etc.)
  lib/            # Shared utilities
```

Database migrations live in `drizzle/` (generated by drizzle-kit, gitignored until populated). Config is in `drizzle.config.ts`.

### TypeScript

All projects use `module: NodeNext` + `moduleResolution: NodeNext`. This means:
- **Relative imports must use `.js` extensions** even for `.ts` source files (e.g. `import './schema.js'`)
- `noUncheckedIndexedAccess` is enabled — use bracket notation for env vars: `process.env['KEY']`

The `packages/typescript-config/node.json` variant (used by `apps/server`) strips DOM libs and disables `declaration`/`declarationMap` since it's an app, not a published package.

### ESLint

- Server uses `@repo/eslint-config/base` (TS + Prettier + Turbo env-var plugin)
- Next.js apps use `@repo/eslint-config/next-js`
- `turbo.json` declares `PORT` and `DATABASE_URL` as known env vars to suppress Turbo ESLint warnings

## About CAGE

CAGE (Confirmed Age, Granted Entry) is an age verification Identity Provider (IdP).
Consumer tagline: "Check Age, Go Everywhere"

### What it does

- Users verify their age once via Veriff (third-party ID verification)
- CAGE stores only the result (age_floor: 18+) — never the raw ID or face scan
- Partner sites (Facebook etc.) integrate via OAuth 2.0 / OIDC
- CAGE issues signed JWT tokens to partners confirming age verification
- Browser extension enables silent token passing on repeat visits

### Critical architecture decisions

- CAGE never stores government IDs, face scans, or exact dates of birth
- Per-partner anonymous sub IDs prevent cross-site user tracking
- Veriff handles all document/biometric processing via their SDK
- Verification results expire after 12 months

### Tech stack

- Server: Hono + TypeScript + Drizzle ORM
- Database: Neon (Postgres) — connection string in Doppler as DATABASE_URL
- Secrets: Doppler (never .env files)
- Monorepo: Turborepo + pnpm

### Database tables

`users`, `verifications`, `partners`, `partner_subs`
